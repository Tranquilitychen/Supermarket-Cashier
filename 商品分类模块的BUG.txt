	原因： 后端生成节点编号的逻辑不完善 （未解决）

3.  显示功能： 
	3.1  list 页面 没有分页因为前端的 tr 与 td 的限制 （想不到 该怎么解决）。（未解决）
	3.2  总条数没有显示
		解决方法：可以采用 model 传出来一个 list 的 size 解决。
	3.3  分类显示的时候不能根据 商品的 级别来排序 
		解决方法：使用Tree 将所有的类型存放在Tree 中，前端解析json进行存放。

4.  删除功能： 
	4.1 已经没有子节点的父节点在页面显示的时候还是会有 左小标 （已解决： 修改 isParent属性） 
	4.2 删除父节点，父节点下的子节点并为删除。
		解决方法： Service中查询父节点下的子节点id,然后进行删除。（已解决： 通过递归）
	4.3 删除节点有商品在使用，出现商品存在，商品节点不存在的情况。(已解决： 删除前查询)
